### 介绍

本仓库是软件工程课程大作业的后端部分。

### 服务架构

#### 1. 用户服务（service-user）
- 职责：
   - 用户注册/登录（支持手机号、邮箱、第三方登录）
   - 用户资料管理（头像、昵称、简介、地理位置等）
   - 用户身份认证与授权（JWT/OAuth2）
   - 用户状态管理（是否被封禁等）
- 数据模型：
- 技术栈：SA-Token + redis

#### 2. 内容服务(service-content)
- 职责：
   - 笔记（内容）的创建、编辑、删除、查询
   - 支持图文、视频、话题标签（#）、@用户
   - 笔记状态管理（草稿、公开、仅自己可见、删除）
   - 内容版本管理（可选）
   - 点赞（Like） 收藏（Favorite） 评论（Comment，支持嵌套）
   - 关注/取关用户
   - 分享
- 数据模型：

#### 3. 搜索服务(service-search)
- 职责：
   - 全文检索（笔记、用户、话题）
   - 搜索关键词建议、热搜榜
- 技术栈：Elasticsearch + Redis

#### 4. 消息与通知服务(service-notification)
- 职责：
   - 系统通知（点赞、评论、关注等）
   - 私信功能
   - 群聊
- 技术栈：WebSocket + RocketMQ + Redis

#### 5. 文件与媒体服务（service-media）
- 职责：
    - 图片/视频上传、存储（OSS/S3）
    - 媒体转码（视频压缩、缩略图生成）
- 技术栈：AliOSS/CloudFlare R2
